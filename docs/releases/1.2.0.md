## 中文

### 后端大改版：从三套接法变成一套 EchoInk Harness

`1.2.0` 重做了 EchoInk 的 Agent 运行层。以前，普通对话、知识库和写作功能各自连接 Codex、OpenCode 或 Hermes，状态、上下文和过程展示容易随后端变化。现在，EchoInk 先把任务交给统一 Harness，再由 Adapter 对接具体 Agent。

| 对比项 | 以前 | 现在 | 直接好处 |
| --- | --- | --- | --- |
| 任务入口 | 对话、知识库和写作各有执行链路 | 统一进入 EchoInk Harness | 停止、超时、失败和完成状态一致 |
| Agent 接入 | 侧栏直接适配不同后端 | Codex、OpenCode、Hermes 通过 Adapter 接入 | 换后端不再换一套交互逻辑 |
| 会话与上下文 | 各链路分别维护 | 统一编译上下文并管理原生会话租约 | 连续对话更稳，减少重复冷启动 |
| 过程展示 | 后端决定侧栏能显示什么 | EchoInk 定义统一过程时间线和结果区 | 不管使用哪个 Agent，阅读顺序和状态口径保持一致 |
| 后续扩展 | 新增后端需要改动多处 UI 和业务逻辑 | 新后端只需实现 Harness Adapter | 更容易增加第四个 Agent，也更容易测试 |

Agent 原生事件的细节仍取决于对应后端是否提供，但 EchoInk 不再为 Hermes 或 OpenCode 使用另一套对话格式，也不会因为切换后端清空当前会话。

### UI 完全重构

这次重构覆盖了整个插件侧栏，不只是换颜色或圆角。

- **对话区：** 最终回答成为视觉主线。思考、命令、文件编辑、MCP 调用和工具状态收进可展开的过程时间线；默认只显示“已处理多久”，展开后再看完整过程。每轮底部保留 Agent、token 和上下文占用。
- **输入区：** 工作区选择移到左上角；Plan 模式会在旁边显示“计划”。收藏、Skill、提示词增强、权限、模型、推理强度和速度改成轻量控件，减少长期占用侧栏空间。
- **参数菜单：** 模型、推理和速度不再平铺成一整行，改为分层菜单，并会根据侧栏边界自动选择展开方向。
- **顶部按钮：** 原来的“活跃”状态改为 Codex / OpenCode / Hermes Agent 切换器。MCP 和设置恢复为透明图标按钮，只在悬停时出现浅灰反馈。
- **知识库频道：** 保留清楚的频道边界，同时与普通对话复用同一套 Agent 过程展示。

### 新增的小功能

- **独立提示词增强：** 侧栏输入框新增 Sparkles 图标。提示词增强拥有自己的 Agent 后端、Provider、API 路径和模型设置，不占用主 Agent 的聊天模型；默认使用内置 WorkBuddy Meta-Prompt，增强后可继续编辑或一键“还原”。
- **独立轻量模型路由：** 改写、扩写、续写和提示词增强都按“小功能任务”调用模型。未手动指定时，优先使用对应后端的快速模型，不会把主 Agent 聊天模型强制改成 Terra 或 Flash。
- **统一收藏入口：** 公众号和公开网页合并为一个“收藏”按钮，粘贴链接后自动判断采集路径。
- **主 Agent 快速切换：** 顶部可随时切换 Codex、OpenCode 和 Hermes；正在运行任务时暂时锁定，避免一轮任务中途换引擎。

### 修复

- 修复消息区上滑查看历史时，被新过程内容强制拉回底部的问题。
- 修复知识库任务状态、报告卡片和滚动抖动。
- 统一失败、中断、取消、断连和五分钟超时的可见状态，减少“后台停了但界面还在等”的情况。
- 修复部分升级路径仍显示旧输入区、缺少独立提示词增强设置或继续使用旧提示词模板的问题。
- 修复窄侧栏下工作区名称、Plan 标记、模型菜单和浮层越界。
- 将提示词增强后的“恢复原文”精简为无边框的“还原”按钮。

### 升级与使用

1. 从本 Release 下载 `main.js`、`manifest.json`、`styles.css`，覆盖 Obsidian 插件目录中的同名文件。
2. 重载 Obsidian 后，在 EchoInk 顶部选择主 Agent。
3. 需要独立设置提示词增强或写作模型时，打开插件设置中的对应页面；不设置则使用自动快速路由。
4. 已有 Vault、EchoInk 会话和自定义模型设置无需迁移或清空。

## English

### Backend redesign: three integrations now share one EchoInk Harness

EchoInk `1.2.0` rebuilds the Agent runtime. Previously, chat, Knowledge, and writing connected to Codex, OpenCode, or Hermes through separate flows, so state handling, context, and process rendering could change with the backend. Tasks now enter one EchoInk Harness before an adapter talks to the selected Agent.

| Area | Before | Now | Benefit |
| --- | --- | --- | --- |
| Task entry | Chat, Knowledge, and writing had separate execution paths | All tasks enter the EchoInk Harness | Stop, timeout, failure, and completion behave consistently |
| Agent integration | The sidebar adapted directly to each backend | Codex, OpenCode, and Hermes connect through adapters | Switching engines no longer changes the interaction model |
| Session and context | Each flow maintained its own state | Context compilation and native-session leases are shared | More stable follow-up turns with fewer unnecessary cold starts |
| Process rendering | Each backend determined what the sidebar showed | EchoInk owns one process timeline and result area | Reading order and status language stay consistent across Agents |
| Extensibility | A new backend required changes across UI and feature code | A new backend implements the Harness adapter contract | A fourth Agent is easier to add and test |

The amount of native event detail still depends on what each backend exposes, but Hermes and OpenCode no longer use a different conversation layout, and switching the main Agent does not clear the current EchoInk session.

### Complete UI rebuild

This release rebuilds the entire sidebar rather than applying a visual skin.

- **Conversation:** Final answers are the primary reading path. Reasoning, commands, file edits, MCP calls, and tool states live in an expandable timeline. The collapsed view shows processing time; the expanded view reveals the full process. Each turn keeps Agent, token, and context usage metadata.
- **Composer:** Workspace selection moves to the upper-left, with a Plan marker beside it when Plan mode is active. Bookmark, Skill, prompt enhancement, permissions, model, reasoning effort, and speed use lighter controls that consume less permanent space.
- **Parameter menus:** Model, reasoning, and speed are grouped into layered menus instead of one long row. Popovers reposition themselves to stay inside narrow sidebars.
- **Header:** The old active-state pill is now a Codex / OpenCode / Hermes Agent switcher. MCP and Settings are transparent icon buttons with a light hover response.
- **Knowledge channel:** The channel keeps a clear boundary while reusing the same EchoInk process projection as ordinary chat.

### Smaller additions

- **Independent prompt enhancement:** The sidebar composer now includes a Sparkles action. Prompt enhancement has its own Agent backend, provider, API path, and model settings, so it does not consume or change the main chat model. It uses the built-in WorkBuddy Meta-Prompt and supports editing or restoring the original input.
- **Independent utility-model routing:** Rewrite, expand, continue, and prompt enhancement run as utility tasks. Automatic routing favors the fast model for the selected utility backend without forcing the main chat Agent to use Terra or Flash.
- **One Bookmark entry:** WeChat articles and public web pages share one Bookmark action that routes each URL automatically.
- **Fast main-Agent switching:** Switch between Codex, OpenCode, and Hermes from the header. The control is temporarily disabled while a task is running so one turn cannot change engines halfway through.

### Fixes

- Fixed message history snapping back to the bottom while the user is reading earlier content.
- Stabilized Knowledge task states, report cards, and scrolling.
- Unified visible failed, interrupted, canceled, disconnected, and five-minute timeout states so the UI does not keep waiting after a backend has stopped.
- Fixed upgrade paths that could still show the old composer, omit the independent prompt-enhancer settings, or keep an outdated prompt template.
- Fixed workspace-name, Plan-marker, model-menu, and popover overflow in narrow sidebars.
- Replaced the long Restore Original label with a concise, borderless Restore action.

### Upgrade and use

1. Download `main.js`, `manifest.json`, and `styles.css` from this Release and replace the matching files in the Obsidian plugin folder.
2. Reload Obsidian, then choose the main Agent from the EchoInk header.
3. Open the corresponding plugin settings page only when you want to override prompt-enhancement or writing models. Otherwise, automatic fast routing is used.
4. Existing Vault files, EchoInk sessions, and custom model settings do not need migration or reset.
